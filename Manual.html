Overview

The course registration system allows students to enroll in courses, view schedules, and manage registrations. The home page is the base of operations for the site, each page has a link back to the home page so that you can do each step of the process as needed. There are six tabs, enroll student, add instructor, register course, add course, drop course, and the manuals.

Web Pages Summary

  • Home Page: Location of all links relevant to registering for classes at Marietta College
  • Enroll Student Page: This page is where a student can enroll for classes and verify that they are eligible.
  • Add Instructor: This page is were you would look at what professors you want for to take or what they are teaching.
  • Add Course: This page is dedicated to searching for available courses based on days of the week a class is available and the time its offered.
  • Register Course: This page is were a student would register for classes.
  • Drop Course: This is were a student would request to drop a class.

Detailed Page Descriptions

Course Registration System Pages

The registration system contains the six pages listed above. Each page has a ton of different fields, but each one is labeled with what is the intended input. The home page is the home base, its where the navigation bar is and the introduction to the site. The next page is the enroll student page, here we have fields for first/last name, major, grade level, and email address. The next page is where an instructor is added, so this page has first/last name, professor rank, department, and email. The third page is for adding a course, and this page has alot of fields for the user, such as the class semester, course number, course name, what time its offered, what days of the week its offered. This page contains alot of information, but its all labeled with what the intended input is. The register course page is similar, but its just the student first/last name, the semester and year of the course, and the course id number. The fifth page is for dropping a course, so fields are course number, semester and time taken, and the students first/last name.

Technical Overview

The system is built using HTML, CSS, JavaScript for frontend, and PHP/MySQL for backend. The current system is only a series of HTML pages

Web Pages Summary

  • index.html: Home page with login functionality.
  • enrollment.html: This page is where a student would enroll.
  • instructor.html: This page is were a student would pick a professor.
  • addcourse.html: This page is dedicated to adding a course to a students schedule.
  • registercourse.html: Course registration page, so this page contains all the course information so the student can add it to their schedule.
  • dropcourse.html Drop course page, this is were you would request to drop a course from your schedule

Page Functions

In this section of the manual, we are going to dive into detail on how the site functions. The pages all follow a similar format, they have input boxes for individual user information. Page by page they are structured the same, the enrollment and instructor page contains fields for first and last names, which are just input boxes. Since theres no backend (for now), some of the inputs will be invalid in the future. For example, the register page and enrollment page need a list of majors and there respective acronyms that are acceptable.

Generally, there are quite a view drop down forms for the user, so the classes are at specific times and taught on specific days. Some of the fields that are provided are left to be unique for the user, with names, email, and major being a few. The forms are all set up the same, with a label for the box and a tag for whats supposed to go there. The drop down menus are also within that form, so its setup in a way that is consistent across the pages.
